# Product Brief: Internal Marketing Performance Briefing Tool

## Problem
The marketing team repeatedly answers the same question: how is marketing performing across channels, and where should we focus next? Today this is done manually by pulling numbers from different tools and stitching them into a response. The result is slow, inconsistent, and dependent on whoever happens to be available.

## Goal
Create a lightweight internal tool that standardizes marketing performance reporting and makes it faster to answer the same question in a clear, repeatable way.

## Primary Users
- Internal analyst
- Account manager / client success team

## Secondary User
- Client, indirectly, through a prepared summary or exported report

## What the Tool Should Do
Version 1 should:
- Let the user choose a client and date range.
- Pull the latest available marketing data from existing systems.
- Show a consistent summary across channels.
- Highlight top-performing and underperforming channels.
- Show changes compared with the previous period.
- Produce a short, readable performance brief.

## Why This Matters
This saves time, reduces manual work, and makes reporting more consistent across the team. It also reduces dependence on a single person who knows how to assemble the answer.

## Success Criteria
A successful tool should help the user quickly answer:
- What happened?
- Which channels are doing well?
- Which channels need attention?
- What changed since the last reporting period?

## Trust Factors
The tool should show:
- Data source names
- Last updated time
- Clear metric definitions
- A consistent output format
- A note when data is incomplete or delayed

## V1 Out of Scope
- Client editing
- Budget optimization
- Forecasting
- Cross-client benchmarking
- Full conversational AI interface
- Replacing existing marketing tools

## Reasoning
These features are intentionally excluded because they add complexity without solving the main problem first. The first version should focus on dependable summaries, not advanced automation.
